- SQLite test compatibility: YEAR()/MONTH()/DAY() are MySQL-only; use driver-aware helpers in services (see DashboardStatsService::yearExpr/monthExpr/dayExpr). Also replace `->whereYear()` with `->whereRaw()`.
- ActivityFactory is at database/factories/ActivityFactory.php — use for tests and seeders.
- .env must use Docker service names as hostnames: DB_HOST=mariadb, REDIS_HOST=redis, MAIL_HOST=mailpit (not 127.0.0.1)
- MariaDB Docker credentials: user=pemiq, password=pemiq, database=pemiq
- User model uses traditional $fillable/$hidden/$casts properties (not PHP 8.4 #[Fillable] attribute syntax)
- Run migrations via: docker compose exec app php artisan migrate
- PHP version: 8.4.22, Laravel: 13.15.0, Filament: v4.11.7
- Filament Color API: use `Color::hex('#rrggbb')` for custom hex colors (not `Color::fromHex()`)
- User model implements FilamentUser interface for panel access control; `canAccessPanel()` checks Spatie role
- Laravel 11: `Registered → SendEmailVerificationNotification` must be explicitly registered in AppServiceProvider::boot()
- Auth views use `resources/views/layouts/auth.blade.php` with Tailwind CDN (no compiled assets needed for auth pages)
- Authenticated app views use `resources/views/layouts/app.blade.php` (navbar + Alpine.js + flash messages)
- Named route 'login' must exist before any auth middleware is applied (redirect target for unauthenticated users)
- Alpine.js CDN: `<script defer src="https://cdn.jsdelivr.net/npm/alpinejs@3.x.x/dist/cdn.min.js">` (defer is required)
- Flash messages: app layout checks session('status'), session('success'), session('error') and $errors
- **Filament v4 BREAKING**: `Filament\Forms\Form` does NOT exist — use `Filament\Schemas\Schema` for form() method
- **Filament v4 BREAKING**: `$navigationGroup` is typed `UnitEnum|string|null` in parent — use `getNavigationGroup(): ?string` method instead of static property
- **Filament v4 BREAKING**: `$navigationIcon` is typed `BackedEnum|string|null` in parent — use `getNavigationIcon()` method instead
- Filament resource discovery needs `php artisan optimize:clear` to pick up newly created resources (not just filament:clear-cached-components)
- ApexCharts integration: import from npm in app.js (window.ApexCharts = ApexCharts), build with `npm run build`, add @vite(['resources/js/app.js']) to layout. DO NOT import Alpine.js in app.js (Livewire 3 bundles Alpine).
- ApexCharts + Livewire: use wire:ignore on chart div, inline <script>window.__data = @json($data);</script> for initial data, $this->dispatch() for updates, Alpine x-init for creation, @event.window for updates.
- npm runs on HOST (not inside Docker container) — use `npm install` and `npm run build` on the host.
- Leaflet CSS in Vite: import in app.js (`import 'leaflet/dist/leaflet.css'`), Vite extracts it and @vite() auto-injects the link tag.
- Non-Livewire pages: init JS using `document.addEventListener('DOMContentLoaded', ...)` to safely access globals set by Vite module scripts (modules execute before DOMContentLoaded).
- i18n in Blade/JS: inline `{{ __('messages.key') }}` inside Alpine.js x-data JS strings is valid — Blade renders server-side before JS is parsed by browser.
- Month names in lang files: `trans('messages.months')` returns 0-indexed array. Use `collect(...)->mapWithKeys(fn($name, $i) => [$i + 1 => $name])` to convert to 1-indexed for month selectors.
- Locale-aware formatting: use `fmt_number($value, $decimals)` (NOT `number_format()`) and `fmt_date($date, $format)` (NOT `->format()`) in all views — defined in `app/Helpers/FormatHelper.php`, autoloaded via `composer.json` files[]. IT uses `,`/`.`; EN uses `.`/`,`.
- SQLite integer division gotcha: `SUM(integer_col) / integer_literal` performs integer division in SQLite. Always use `/ 1000.0` or `/ 3600.0` (float literals) to force float division. DashboardStatsService has this issue too but existing tests happen to use multiples that divide evenly.
- TrendAnalysisService weekly/monthly period labels: SQLite returns `strftime('%Y-%W', ...)` → "2024-03", MySQL `DATE_FORMAT('%Y-%u')` → same "YYYY-WW" format. Monthly: `strftime('%Y-%m', ...)` and `DATE_FORMAT('%Y-%m')` both return "2024-01" format.
